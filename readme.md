# 🏢 Back-Office History Management System (BOMS)

**"흩어진 업무 히스토리를 하나의 타임라인으로, AI 기반 스마트 백오피스 솔루션"**

## 📝 1. 프로젝트 개요 (Overview)
실무 중 과거의 결정 사항이나 히스토리를 파악하기 위해 메일함과 협업 툴을 전전해야 했던 비효율적인 경험을 해결하기 위해 시작되었습니다.

**BOMS**는 1~50인 규모의 중소기업을 위한 최적화된 업무 관리 플랫폼입니다. 단순한 업무 기록을 넘어, AI 분석을 통해 사내 주요 이슈를 자동으로 뉴스화하고 자연어 검색으로 복잡한 히스토리를 단숨에 찾아낼 수 있는 환경을 제공합니다. 이를 통해 업무의 투명성을 높이고 구성원의 성과 관리를 데이터 기반으로 지원합니다.

## 🎯 2. 기획 배경 및 사용 대상 (Background & Target)
### 2.1 기획 배경
* **히스토리 파악의 병목 현상**: 과거 이력을 찾기 위해 소모되는 시간을 단축하여 실질적인 업무 효율성 개선
* **체계적인 업무 분리**: 담당자, 중요도, 진행 여부를 명확히 하여 업무 누락 방지
* **객관적인 성과 관리**: 기록된 타임라인을 통해 개인별 기여도와 성과를 손쉽게 증빙

### 2.2 주요 사용 대상
* **성장하는 중소기업**: 1~50명 규모의 조직에서 체계적인 업무 관리 시스템이 필요한 경우
* **CS 및 운영 부서**: 고객 요청과 내부 업무 대응 과정을 실시간으로 추적하고 싶은 팀
* **관리가 필요한 리더**: 팀의 전체적인 업무 흐름과 주요 이슈를 AI 요약으로 빠르게 파악하고 싶은 관리자

## ✨ 3. 핵심 기능 (Key Features)
* **📥 CS 요청 및 통합 응답 관리**: 고객의 요청 사항을 접수하고 답변 과정을 체계적으로 관리
* **📊 업무 분리 및 할당**: 담당자 지정, 업무 중요도 및 진행 상태(Status) 관리 기능
* **⏳ 업무 타임라인 (Timeline)**:
    * **대상별 타임라인**: 특정 프로젝트나 고객사에 대한 모든 작업 이력 시각화
    * **요청별 타임라인**: 특정 요청(Request)에 의해 파생된 모든 작업의 연결 고리 제공
* **🗞️ AI 기반 사내 뉴스 생성**: CS 요청 및 업무 데이터를 학습하여 주요 이슈를 자동 요약 및 공유
* **🔍 AI 기반 타임라인 검색**: 자연어 질문으로 과거의 복잡한 업무 히스토리에서 필요한 정보 검색

## 🛠 4. 기술 스택 (Tech Stack)
### Backend
* **Language**: Java 17
* **Framework**: Spring Boot 3.4.12
* **Build Tool**: Gradle 8.14.3
* **DATABASE**: Mysql 9.4
* **ORM**:
    * **Spring Data JPA**: 객체 지향적인 데이터 모델 관리
    * **QueryDSL**: 복잡한 타임라인 검색 및 동적 쿼리 처리
* **Security**: Spring Security를 통한 권한 관리

### Frontend
* **Template Engine**: Thymeleaf
* **Script**: JavaScript
